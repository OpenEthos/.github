# Support

Thank you for using OpenEthos! We're here to help you get the most out of our projects.

## Getting Help

### Documentation

Before reaching out, please check our documentation:

- **Project README**: Each repository has a comprehensive README with getting started guides
- **Wiki**: Check the repository wiki for detailed documentation
- **API Documentation**: Available in the docs/ folder or hosted documentation site
- **Examples**: Look for examples/ directory in the repository

### Community Support

We offer several channels for community support:

#### GitHub Discussions

The best place for questions, ideas, and community interaction:

- **Q&A**: Ask questions and get help from the community
- **Show and Tell**: Share what you've built
- **Ideas**: Propose new features or improvements
- **General**: General discussions about the project

[Visit GitHub Discussions](https://github.com/orgs/OpenEthos/discussions)

#### GitHub Issues

For bug reports and feature requests:

- **Bug Reports**: Report bugs using our bug report template
- **Feature Requests**: Suggest new features or enhancements
- **Security Issues**: Report security concerns (use Security Advisories for vulnerabilities)

[Create an Issue](https://github.com/OpenEthos/.github/issues/new/choose)

### What to Include

When asking for help, please include:

- **Clear Description**: Describe your issue or question clearly
- **Environment Details**:
  - Operating System and version
  - Python version
  - Package version
  - Relevant configuration
- **Steps to Reproduce**: For bugs, include steps to reproduce
- **Expected vs Actual**: What you expected and what actually happened
- **Code Samples**: Minimal code examples that demonstrate the issue
- **Error Messages**: Complete error messages and stack traces
- **What You've Tried**: Solutions you've already attempted

### Response Times

We're an open-source project maintained by volunteers, so response times may vary:

- **Critical Issues**: We aim to respond within 48 hours
- **General Questions**: Usually within a few days
- **Feature Requests**: May take longer to evaluate

## How to Ask Good Questions

To get the best help quickly:

1. **Search First**: Check if your question has been answered
2. **Be Specific**: Provide concrete details, not vague descriptions
3. **Minimal Example**: Create a minimal reproducible example
4. **Show Your Work**: Explain what you've tried
5. **One Question Per Issue**: Don't combine multiple unrelated questions
6. **Follow Up**: Provide feedback and close issues when resolved
